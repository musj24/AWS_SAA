# 기본 용어
------------------------
- 컴퓨팅 : 프로그램을 실행시키는 처리 능력, 서버 장비의 CPU, RAM, OS 등을 클라우드에서 빌려 쓰는 영역
- 종량제 : 리소스를 사용한 만큼만 비용을 지불함

- 클라우드 배포 유형
  - 퍼블릭 클라우드 : AWS, Azure 같이 리소스를 클라우드 서버로 마이그레이션 하는 방식. 클라우드에서 서버, DB, 애플리케이션 구축
  - 프라이빗 : 전용 리소스를 사용하는 방식. (같은 개념은 아니지만 온프레미스도 알아둬야 함)
  - 하이브리드 : 둘 이상의 퍼블릭 클라우드 사업자에게서 서비스를 함께 제공 받는 유형
  
- AWS 글로벌 인프라
    - 고가용성 : 서비스 중단 시간을 최소화 한다는 개념
    - 내결함성 : 구성요소 실패시 지속적인 서비스가 가능하게 하는 설계 (이중화 등)

- 리전 : 데이터 센터를 구축해 놓은 지리적 거점, 3개 이상의 AZ(가용영역) 으로 구성

- AWS 공동 책임 모델
  - AWS는 클라우드의 보안 (하드웨어 및 네트워크 가상화)
  - 고객은 클라우드 내부 보안 (고객 데이터 및 OS)

- Amazon EC2 : amazon elastic cloud computing
  - EC2 instance : AWS 클라우드에서 실행되는 가상 서버. CPU, 메모리, 스토리지, 네트워킹 기능
    - 인스턴스 패밀리
      - 범용 : 균형잡힌 리소스
      - 컴퓨팅 최적화 : 고성능, 기계학습 과학모델
      - 메모리 최적화 : 데이터 처리 
      - 가속 컴퓨팅 : 그래픽처리 
      - 스토리지 최적화 : 데이터 집약 및 저장

- 멀티테넌시 : 기본 하드웨어를 가상 머신들간에 공유
- 수직적 스케일링
- 리소스 프로비저닝
  - API (콘솔, CLI, SDK) 사용
  - 콘솔 : GUI
  - CLI : 스크립트 및 자동화 가능
  - SDK : 다른 프로그래밍 언어로 API 호출

- AMI : Amazon Machine Image
  - 인스턴스를 시작하는데 필요한 기본 구성 요소가 포함된 가상 이미지
